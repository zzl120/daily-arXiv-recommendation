### [1] [Bridge-RAG: An Abstract Bridge Tree Based Retrieval Augmented Generation Algorithm With Cuckoo Filter](https://arxiv.org/abs/2603.26668)
*Zihang Li,Wenjun Liu,Yikun Zong,Jiawen Tao,Siying Dai,Songcheng Ren,Zirui Liu,Yanbing Jiang,Tong Yang*

Main category: cs.IR

TL;DR: Bridge-RAG 是一个新颖的检索增强生成框架，通过引入抽象概念作为查询实体和文档块之间的桥梁，并采用改进的 Cuckoo Filter 数据结构，同时解决了检索精度和计算效率两大挑战。该框架在 MedQA 和 AALCR 数据集上实现了约 15.65% 的准确率提升和显著的检索速度优化。


<details>
  <summary>Details</summary>
Motivation: 现有的 RAG 框架面临两个关键挑战：1) 检索精度方面，基于向量相似性搜索的标准 RAG 倾向于检索语义相关但上下文碎片化或嘈杂的段落，导致生成器接收到不充分或不相关的证据；2) 计算效率方面，结构化 RAG 框架（特别是 Tree-RAG）在数据集规模和树深度增长时，遍历层次结构和定位相关实体的时间显著增加。这两个挑战阻碍了 RAG 系统在对精度和低延迟响应有高要求的实际场景中的部署。

Method: Bridge-RAG 采用两个关键设计：1) 为提升检索精度，引入抽象概念将连续文档块分组为高级知识单元，组织成树结构，并设计多级检索策略；2) 为提升效率，引入改进的 Cuckoo Filter 数据结构，支持 O(1) 时间复杂度的实体查找，采用块链表结构和基于实体温度的排序机制优化空间和时间局部性。具体流程包括实体识别、Cuckoo Filter 查找、抽象集扩展、文档块检索和相似度排序五个步骤。

Result: 在 MedQA 和 AALCR 数据集上的实验表明，Bridge-RAG 相比基线方法有显著改进：在 AALCR 数据集上准确率提升约 22.1%，在 MedQA 数据集上提升约 15.65%。检索时间方面，Bridge-RAG 的 Cuckoo Filter 实现 O(1) 查找复杂度，相比 Tree-RAG 的广度优先搜索遍历大幅提升效率。多级深度设置（Depth=1,2,3）的实验显示了精度和效率之间的权衡，深度增加带来更丰富的上下文但略微增加检索时间。

Conclusion: Bridge-RAG 通过将层次抽象树与改进的 Cuckoo Filter 相结合，同时提高了检索增强生成的准确性和降低了延迟。通过追踪多个抽象级别的父子关系，模型获得了高级概念和细粒度证据，而块链表的 Cuckoo Filter 保证了 O(1) 的实体到抽象查找，并通过自动重新排序热实体维持持续速度。实验证明了该方法在医学问答和学术文献检索任务上的有效性。

Abstract: As an important paradigm for enhancing the generation quality of Large Language Models (LLMs), retrieval-augmented generation (RAG) faces the two challenges regarding retrieval accuracy and computational efficiency. This paper presents a novel RAG framework called Bridge-RAG. To overcome the accuracy challenge, we introduce the concept of abstract to bridge query entities and document chunks, providing robust semantic understanding. We organize the abstracts into a tree structure and design a multi-level retrieval strategy to ensure the inclusion of sufficient contextual information. To overcome the efficiency challenge, we introduce the improved Cuckoo Filter, an efficient data structure supporting rapid membership queries and updates, to accelerate entity location during the retrieval process. We design a block linked list structure and an entity temperature-based sorting mechanism to improve efficiency from the aspects of spatial and temporal locality. Extensive experiments show that Bridge-RAG achieves around 15.65% accuracy improvement and reduces 10x to 500x retrieval time compared to other RAG frameworks.

### [2] [ReCQR: Incorporating conversational query rewriting to improve Multimodal Image Retrieval](https://arxiv.org/abs/2603.26669)
*Yuan Hu,ZhiYu Cao,PeiFeng Li,QiaoMing Zhu*

Main category: cs.IR

TL;DR: 本文提出了ReCQR数据集，这是首个用于图像检索中对话式查询重写的基准数据集，包含约7,000个高质量多模态对话。通过将对话式查询重写任务引入图像检索领域，将用户模糊、上下文依赖的查询重写为简洁、语义完整的检索友好查询，显著提升了传统图像检索模型的准确性。


<details>
  <summary>Details</summary>
Motivation: 随着多模态学习的发展，图像检索在连接视觉信息与自然语言查询中扮演关键角色。现有图像检索器在处理长文本和模糊用户表达方面存在困难。特别是在对话式场景中，用户查询往往具有上下文依赖性且语义不完整，包含无法脱离对话历史解析的指代信息。虽然CLIP等模型在单轮检索中表现出色，但在对话式设置中难以解析这些指代。现有方法要么编码整个对话历史引入噪声和冗余，要么缺乏针对图像检索性能的优化。因此，需要将文本领域已证明有效的对话式查询重写技术引入多模态图像检索，以弥合先进但静态的视觉语言模型与动态多模态对话之间的鸿沟。

Method: 研究采用两阶段方法：1）数据集构建：基于MSCOCO语料库，使用两阶段流水线构建ReCQR数据集。第一阶段生成仅文本对话，包含需要历史解析的上下文依赖查询；第二阶段引入多模态对话，用户引用多个图像，创建跨模态依赖。使用大型语言模型（Qwen2.5-VL-7B-Instruct、Qwen3-14B）生成改写候选，结合LLM-as-Judge机制和人工审核筛选高质量数据。2）实验评估：采用两阶段训练评估流水线，第一阶段仅使用对话历史进行文本重写，第二阶段加入视觉上下文进行多模态重写。评估了三种多模态大语言模型（Qwen2.5-VL-7B-Instruct、LLaVA-v1.6-Mistral-7B-HF、GLM-4.1V-9B-Thinking）作为查询重写模型，使用CLIP作为固定检索骨干模型。主要评估指标为Recall@K（R@1、R@5、R@10）。

Result: 实验结果显示：1）对话式查询重写显著提升了检索性能，原始查询与目标查询之间存在显著性能差距（如文本仅设置中R@1从3.6%提升至22.4%）；2）在ReCQR数据集上的微调相比基础模型的零样本能力带来实质性性能提升；3）多模态数据集相比文本仅数据集具有显著更高的复杂性，所有模型在多模态设置下表现更差；4）多模态训练对处理跨图像引用至关重要，多模态版本模型显著优于仅文本版本；5）不同模型在不同指标上表现优异：GLM-4.1V-9B-Thinking在单图像文本仅设置中达到最高R@1分数（15.4%），而LLaVA-v1.6-Mistral-7B-HF在多模态任务中表现出色（R@10达32.2%）。

Conclusion: 本文成功将对话式查询重写引入多模态图像检索领域，构建了首个专门的数据集ReCQR。实验证明，查询重写能够有效弥合模糊的多轮查询与检索就绪表示之间的差距，显著提升检索性能。这项工作为未来多模态对话系统中用户查询建模提供了新的方向和见解，展示了利用现成检索器处理复杂多模态对话的可行性。

Abstract: With the rise of multimodal learning, image retrieval plays a crucial role in connecting visual information with natural language queries. Existing image retrievers struggle with processing long texts and handling unclear user expressions. To address these issues, we introduce the conversational query rewriting (CQR) task into the image retrieval domain and construct a dedicated multi-turn dialogue query rewriting dataset. Built on full dialogue histories, CQR rewrites users' final queries into concise, semantically complete ones that are better suited for retrieval. Specifically, We first leverage Large Language Models (LLMs) to generate rewritten candidates at scale and employ an LLM-as-Judge mechanism combined with manual review to curate approximately 7,000 high-quality multimodal dialogues, forming the ReCQR dataset. Then We benchmark several SOTA multimodal models on the ReCQR dataset to assess their performance on image retrieval. Experimental results demonstrate that CQR not only significantly enhances the accuracy of traditional image retrieval models, but also provides new directions and insights for modeling user queries in multimodal systems.

### [3] [SRAG: RAG with Structured Data Improves Vector Retrieval](https://arxiv.org/abs/2603.26670)
*Shalin Shah,Srikanth Ryali,Ramasubbu Venkatesh*

Main category: cs.IR

TL;DR: 本文提出了结构化RAG（Structured RAG）方法，通过在查询和文本块中添加结构化元数据（包括主题、情感、查询类型、知识图谱三元组和语义标签）来改进传统检索增强生成（RAG）系统。该方法显著提升了检索质量，在金融问答任务中使LLM-as-a-Judge评分提高了30%，特别是在分析性、比较性和预测性查询上表现突出。


<details>
  <summary>Details</summary>
Motivation: 传统RAG系统仅依赖于查询和文本块之间的向量表示相似性进行检索，这存在局限性：1）检索仅基于表面层面的表示相似性，忽略了语义和结构信息；2）容易导致过度过滤，仅检索与查询在词汇上相似的内容；3）无法有效支持情景式检索（episodic retrieval），即无法重新激活先前经验中的潜在信息；4）在分析性、比较性和预测性查询上表现不佳。研究旨在通过添加结构化元数据来改善检索表示，从而提升RAG系统的性能。

Method: 结构化RAG方法的核心是在索引前为文本块和查询添加结构化元数据，包括：1）主题（Topics）：如收入增长、收益估计、市场表现等；2）情感（Sentiments）：积极、消极或中性；3）查询和文本块类型（Query and Chunk Classes）：如定量型、比较型、分析型、信息查找型、信息型等；4）知识图谱三元组（KG Triples）：结构化事实信息；5）语义标签（Semantic Tags）：如财务指标、估值参数等。该方法不需要改变底层系统架构，只需重新分块和标记，然后将标记后的查询和文本块用于向量检索。检索到的标记文本块被添加到提示中，由LLM合成答案。

Result: 实验结果显示：1）在所有查询类型上，结构化RAG（平均得分94.35）显著优于普通RAG（平均得分72.36），p值为2e-13；2）在特定查询类型上提升尤为显著：预测性查询（95.61 vs 64.46，p=9e-5）、分析性查询（93.8 vs 65.1，p=2e-5）、比较性查询（94.1 vs 55.9，p=0.0002）；3）信息查找型查询两者表现相似（97.43 vs 98.37，p=0.24）；4）使用GPT-5作为LLM-as-a-Judge评估，结构化RAG使答案评分提高了30%；5）消融研究表明，性能提升是多个元数据组件共同作用的结果，而非单一特征主导，其中语义标签、主题和文本块类型的影响最大。

Conclusion: 结构化RAG是一种简单实用的RAG扩展方法，通过添加结构化元数据显著改善了检索质量。该方法不需要架构改变，易于集成到现有RAG管道中。实验证明，结构化RAG在金融问答任务中大幅提升了性能，特别是在需要推理的查询类型上。该方法通过促进更广泛、更多样化的检索，可能支持情景式检索，从而改善LLM的泛化能力。未来的工作可以探索更原则化的特征贡献分析方法，并将评估扩展到更多领域。

Abstract: Retrieval Augmented Generation (RAG) provides the necessary informational grounding to LLMs in the form of chunks retrieved from a vector database or through web search. RAG could also use knowledge graph triples as a means of providing factual information to an LLM. However, the retrieval is only based on representational similarity between a question and the contents. The performance of RAG depends on the numeric vector representations of the query and the chunks. To improve these representations, we propose Structured RAG (SRAG), which adds structured information to a query as well as the chunks in the form of topics, sentiments, query and chunk types (e.g., informational, quantitative), knowledge graph triples and semantic tags. Experiments indicate that this method significantly improves the retrieval process. Using GPT-5 as an LLM-as-a-judge, results show that the method improves the score given to answers in a question answering system by 30% (p-value = 2e-13) (with tighter bounds). The strongest improvement is in comparative, analytical and predictive questions. The results suggest that our method enables broader, more diverse, and episodic-style retrieval. Tail risk analysis shows that SRAG attains very large gains more often, with losses remaining minor in magnitude.
