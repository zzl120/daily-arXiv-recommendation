### [1] [Sparton: Fast and Memory-Efficient Triton Kernel for Learned Sparse Retrieval](https://arxiv.org/abs/2603.25011)
*Thong Nguyen,Cosimo Rulli,Franco Maria Nardini,Rossano Venturini,Andrew Yates*

Main category: cs.IR

TL;DR: 本文提出了Sparton，一种针对学习稀疏检索（LSR）模型中语言模型头（LM head）优化的Triton内核。通过融合矩阵乘法、ReLU、log1p和最大池化操作，并采用在线最大归约技术，Sparton避免了中间对数矩阵的完整内存存储，实现了高达4.8倍的加速和数量级的内存使用减少。


<details>
  <summary>Details</summary>
Motivation: 当前最先进的学习稀疏检索模型（如Splade）使用语言模型头将隐藏状态投影到词汇表锚定的对数矩阵，然后通过逐元素操作和序列维度上的最大池化转换为稀疏表示。然而，由于词汇表规模庞大（30,000-250,000个token），这个中间矩阵会创建巨大的内存瓶颈（可达16GB），限制了模型扩展。标准PyTorch实现中，该矩阵在HBM和SRAM之间反复传输，导致显著的I/O开销和运行时性能下降。

Method: Sparton采用融合方法，将分块矩阵乘法、ReLU、log1p和最大归约集成到单个GPU内核中。关键技术包括：1）利用操作单调性重新排序，将最大池化提前到激活函数之前；2）在原始对数分块上执行早期在线归约，避免完整对数矩阵的内存存储；3）采用混合设计，使用cuBLAS/rocBLAS计算词汇表分块的对数，然后立即应用Triton内核进行掩码最大归约；4）在反向传播中，仅存储最大值和索引，通过稀疏梯度计算减少内存带宽需求。

Result: 实验结果表明：1）单独内核测试中，Sparton相比PyTorch基线实现了高达4.8倍的加速和数量级的峰值内存使用减少；2）集成到Splade-v3中，使训练吞吐量提升33%，内存使用减少；3）能够处理序列长度达8192的输入，而基线方法在4096或8192时出现内存不足错误；4）在真实LSR训练中，保持相同检索效果的同时，训练速度提升14%，内存使用减少23%；5）在多语言骨干网络中，支持26%更大的批次大小和2.5倍的训练加速。

Conclusion: Sparton通过融合操作、在线归约和稀疏梯度计算，有效解决了LSR模型中LM head的内存瓶颈问题。它不仅显著提升了训练效率，还支持处理更大的批次大小、序列长度和词汇表规模，为学习稀疏检索模型的扩展提供了关键技术。未来的工作可以探索利用Tensor Memory Accelerator或低精度格式进一步优化现代GPU性能。

Abstract: State-of-the-art Learned Sparse Retrieval (LSR) models, such as Splade, typically employ a Language Modeling (LM) head to project latent hidden states into a lexically-anchored logit matrix. This intermediate matrix is subsequently transformed into a sparse lexical representation through element-wise operations (ReLU, Log1P) and max-pooling over the sequence dimension. Despite its effectiveness, the LM head creates a massive memory bottleneck due to the sheer size of the vocabulary (V), which can range from 30,000 to over 250,000 tokens in recent models. Materializing this matrix creates a significant memory bottleneck, limiting model scaling. The resulting I/O overhead between operators further throttles throughput and runtime performance. In this paper, we propose Sparton, a fast memory-efficient Triton kernel tailored for the LM head in LSR models. Sparton utilizes a fused approach that integrates the tiled matrix multiplication, ReLU, Log1P, and max-reduction into a single GPU kernel. By performing an early online reduction directly on raw logit tiles, Sparton avoids materializing the full logit matrix in memory. Our experiments demonstrate that the Sparton kernel, in isolation, achieves up to a 4.8x speedup and an order-of-magnitude reduction in peak memory usage compared to PyTorch baselines. Integrated into Splade (|V| ~ 30k), Sparton enables a 33% larger batch size and 14% faster training with no effectiveness loss. On a multilingual backbone (|V| ~ 250k), these gains jump to a 26x larger batch size and 2.5x faster training.

### [2] [Hyena Operator for Fast Sequential Recommendation](https://arxiv.org/abs/2603.25027)
*Jiahao Liu,Lin Li,Zhiyuan Li,Kaixi Hu,Kaize Shi,Jingling Yuan*

Main category: cs.IR

TL;DR: HyenaRec是一种新颖的顺序推荐模型，通过结合基于勒让德正交多项式的卷积核参数化和门控机制，在保持线性复杂度的同时有效建模长用户序列中的多尺度行为依赖关系，显著超越了注意力、循环神经网络等基线模型在排序准确性和训练效率方面的表现。


<details>
  <summary>Details</summary>
Motivation: 当前基于注意力的顺序推荐模型虽然准确性高，但存在二次复杂度问题，在处理长用户历史记录时计算成本过高。Hyena等次二次算子虽然在语言建模中表现出色，但在推荐系统中面临挑战：1）在稀疏的长用户序列上表示能力有限；2）难以同时捕捉缓慢演化的长期兴趣和频繁出现的短期行为爆发。需要一种既能高效处理长序列又能稳定建模稀疏反馈下多尺度用户行为的解决方案。

Method: HyenaRec采用混合架构设计：1）使用勒让德正交多项式参数化卷积核，提供平滑紧凑的基函数来建模长期时间依赖关系；2）引入门控线性单元（GLU）机制捕捉细粒度的短期行为爆发；3）构建层次化的Hyena算子堆叠，结合短路径（轻量级卷积和自适应门控）和长路径（基于勒让德多项式的深度可分离卷积）；4）采用因果归一化技术确保信号传播稳定性；5）基于能量准则选择多项式基函数数量（K=64），平衡表达能力和计算效率。

Result: 在ML-1M、Steam、Video和XLong四个真实数据集上的实验表明：1）HyenaRec在NDCG@10、Recall@10等排序指标上一致优于所有基线模型（GRU4Rec、NARM、SASRec、BERT4Rec、EulerFormer、HSTU、LRURec）；2）在长序列场景（XLong）上相对性能提升最为显著（NDCG@10提升17.65%）；3）训练效率显著提高，SASRec和BERT4Rec分别需要1.77×和4.25×的训练时间；4）消融实验验证了多项式核和门控机制的关键作用；5）模型对超参数（dropout、权重衰减）和勒让德滤波器阶数K表现出良好的鲁棒性。

Conclusion: 基于勒让德正交多项式的卷积核参数化为顺序推荐提供了一种原则性且可扩展的注意力替代方案。HyenaRec通过结构化卷积算子和正交多项式参数化的结合，在保持次二次计算复杂度的同时，有效解决了长用户序列建模中的稀疏反馈和多尺度行为依赖挑战，为工业级大规模顺序推荐提供了高效稳定的解决方案。

Abstract: Sequential recommendation models, particularly those based on attention, achieve strong accuracy but incur quadratic complexity, making long user histories prohibitively expensive. Sub-quadratic operators such as Hyena provide efficient alternatives in language modeling, but their potential in recommendation remains underexplored. We argue that Hyena faces challenges in recommendation due to limited representation capacity on sparse, long user sequences. To address these challenges, we propose HyenaRec, a novel sequential recommender that integrates polynomial-based kernel parameterization with gated convolutions. Specifically, we design convolutional kernels using Legendre orthogonal polynomials, which provides a smooth and compact basis for modeling long-term temporal dependencies. A complementary gating mechanism captures fine-grained short-term behavioral bursts, yielding a hybrid architecture that balances global temporal evolution with localized user interests under sparse feedback. This construction enhances expressiveness while scaling linearly with sequence length. Extensive experiments on multiple real-world datasets demonstrate that HyenaRec consistently outperforms Attention-, Recurrent-, and other baselines in ranking accuracy. Moreover, it trains significantly faster (up to 6x speedup), with particularly pronounced advantages on long-sequence scenarios where efficiency is maintained without sacrificing accuracy. These results highlight polynomial-based kernel parameterization as a principled and scalable alternative to attention for sequential recommendation.

### [3] [AuthorityBench: Benchmarking LLM Authority Perception for Reliable Retrieval-Augmented Generation](https://arxiv.org/abs/2603.25092)
*Zhihui Yao,Hengran Zhang,Keping Bi*

Main category: cs.IR

TL;DR: 本文提出了AuthorityBench，一个用于评估大语言模型权威感知能力的综合基准，包含DomainAuth、EntityAuth和RAGAuth三个数据集。研究评估了五种LLM在三种判断方法下的表现，发现ListJudge和PairJudge结合PointScore输出与真实权威相关性最强，而ListJudge具有最佳成本效益。研究还发现包含网页文本会降低判断性能，表明权威性不同于文本风格。下游RAG实验显示权威引导的过滤能显著提高答案准确性。


<details>
  <summary>Details</summary>
Motivation: 检索增强生成（RAG）虽然增强了LLM的外部知识获取能力，但仍容易受到低权威来源的影响，从而传播错误信息。现有研究主要关注文档的相关性和实用性，但忽略了信息来源的权威性这一关键维度。权威性作为内容无关的、基于发布者或实体的先验属性，反映了其结构性影响或全球认可度。本研究旨在系统性地探究LLM是否能够感知权威信号，并评估这种能力在RAG系统中的实际价值。

Method: 研究构建了AuthorityBench基准，包含三个数据集：DomainAuth（基于10,000个网页域的PageRank权威评分）、EntityAuth（基于22,000个实体的Wikipedia跨语言链接数）和RAGAuth（120个是/否问题及其检索文档）。评估了五种LLM（Qwen3-8B/14B/32B、Llama-3-8B-Instruct、Llama-3.1-8B-Instruct）在三种判断方法下的表现：PointJudge（独立评分）、PairJudge（成对比较）和ListJudge（列表排序）。每种方法支持多种输出格式（ListRank、PointScore），并在有/无网页文本的两种设置下进行测试。下游RAG实验比较了基于权威性、相关性和实用性的文档过滤策略。

Result: 1. ListJudge和PairJudge结合PointScore输出在所有设置中与真实权威标签的相关性最强（Spearman's ρ和Kendall's τ最高）。
2. PairJudge计算成本最高，而ListJudge在成本效益方面表现最佳。
3. 包含网页文本通常会降低LLM的判断性能，表明权威性不同于文本风格、流畅性或叙述丰富性。
4. 模型规模与权威感知能力呈正相关，更大的模型表现更好。
5. 在RAGAuth实验中，基于权威性的过滤策略在所有模型中实现了最高的答案准确率提升（例如Qwen3-14B从58.33%提升到66.67%），显著优于基于相关性和实用性的过滤。
6. 实体权威性（EntityAuth）的感知通常比域权威性（DomainAuth）更强且更一致，特别是在篮球和电影领域。

Conclusion: LLM确实具备感知权威性的能力，但这种能力因判断方法和模型规模而异。ListJudge和PairJudge结合PointScore是最有效的权威评估方法，而ListJudge提供了最佳的成本效益平衡。权威感知在RAG系统中具有重要实际价值，基于权威性的文档过滤能显著提高答案准确性并减少错误信息的误导。本研究为理解和利用LLM的权威感知能力提供了系统基准，是构建更可靠和可信AI系统的重要一步。

Abstract: Retrieval-Augmented Generation (RAG) enhances Large Language Models (LLMs) with external knowledge but remains vulnerable to low-authority sources that can propagate misinformation. We investigate whether LLMs can perceive information authority - a capability extending beyond semantic understanding. To address this, we introduce AuthorityBench, a comprehensive benchmark for evaluating LLM authority perception comprising three datasets: DomainAuth (10K web domains with PageRank-based authority), EntityAuth (22K entities with popularity-based authority), and RAGAuth (120 queries with documents of varying authority for downstream evaluation). We evaluate five LLMs using three judging methods (PointJudge, PairJudge, ListJudge) across multiple output formats. Results show that ListJudge and PairJudge with PointScore output achieve the strongest correlation with ground-truth authority, while ListJudge offers optimal cost-effectiveness. Notably, incorporating webpage text consistently degrades judgment performance, suggesting authority is distinct from textual style. Downstream experiments on RAG demonstrate that authority-guided filtering largely improves answer accuracy, validating the practical importance of authority perception for reliable knowledge retrieval. Code and benchmark are available at: https://github.com/Trustworthy-Information-Access/AuthorityBench.

</details>


### [4] [Supercharging Federated Intelligence Retrieval](https://arxiv.org/abs/2603.25374)
*Dimitris Stripelis,Patrick Foley,Mohammad Naseri,William Lindskog-Münzing,Chong Shen Ng,Daniel Janes Beutel,Nicholas D. Lane*

Main category: cs.IR

TL;DR: 本文提出了一种基于Flower框架的安全联邦RAG系统，通过本地文档检索和服务器端在可信执行环境中的聚合与文本生成，实现了跨私有数据孤岛的机密知识检索与生成，同时引入了级联推理机制整合第三方模型作为辅助上下文。


<details>
  <summary>Details</summary>
Motivation: 传统RAG系统假设文档集中访问，但在实际应用中，组织数据通常分布在多个私有数据孤岛中，由于监管、专有性或隐私限制而无法集中聚合。现有联邦RAG方法在安全方面存在局限，检索到的文档以明文形式交换，可能暴露给诚实但好奇或已遭破坏的服务器。因此需要一种能够保护联邦聚合和生成过程的安全FedRAG架构。

Method: 1. 基于Flower框架构建安全联邦RAG系统，在客户端本地执行文档检索（使用FAISS索引），服务器端在可信执行环境中进行聚合和文本生成。2. 采用级联推理方法，整合非机密的第三方模型作为辅助概率知识源，不损害端到端机密性。3. 使用Flower Confidential Remote Compute服务实现受保护的远程LLM推理。4. 通过硬件强制的TEE和远程认证机制确保安全性，假设服务器操作者和基础设施提供商可能是诚实但好奇或已遭破坏的。

Result: 1. 级联推理模式（使用轻量级Amazon Nova Micro模型）相比独立推理模式，在PubMedQA数据集上准确率提升40%。2. 机密推理模式（使用Qwen3 275B模型在Flower CRC中）在所有评估数据集上获得最高准确率和最低延迟：PubMedQA准确率0.47，延迟25秒；BioASQ准确率0.82，延迟26秒；MedQA准确率0.78，延迟25秒。3. CPU绑定的SmolLM生成在独立和级联模式中主导延迟，而机密推理受益于在Flower CRC的强化TEE中直接接收加密聚合文档。

Conclusion: 提出的安全联邦RAG架构在保护数据局部性的同时，通过机密计算保护了聚合和生成过程。实验表明，使用轻量级非机密第三方模型进行概率上下文增强的级联推理提高了性能，突显了CPU基TEE在混合信任部署中的实用性。利用强化TEE中的高容量GPU支持模型，机密推理实现了更高的准确率和更低的延迟，支持了灵活、保护隐私的RAG设计。

Abstract: RAG typically assumes centralized access to documents, which breaks down when knowledge is distributed across private data silos. We propose a secure Federated RAG system built using Flower that performs local silo retrieval, while server-side aggregation and text generation run inside an attested, confidential compute environment, enabling confidential remote LLM inference even in the presence of honest-but-curious or compromised servers. We also propose a cascading inference approach that incorporates a non-confidential third-party model (e.g., Amazon Nova) as auxiliary context without weakening confidentiality.

</details>
