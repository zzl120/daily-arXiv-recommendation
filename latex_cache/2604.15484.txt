All code, data, and experiments are open-source. {0.5pt} Large language model agents increasingly require persistent memory -- the ability to store, retrieve, and prioritize information across sessions. While cloud-hosted vector databases serve this need at scale, many use cases demand local-first operation: developer tooling, personal knowledge management, privacy-sensitive workflows, and offline agents. Existing local solutions face three gaps: Pure vector search misses exact keywords ; pure keyword search misses semantic paraphrases. Hybrid fusion helps, but combining scores from incompatible distributions is non-trivial. Documents accessed yesterday should rank differently from documents untouched for months. Most RAG systems treat all chunks equally regardless of usage history. When every query returns results with uniformly high scores, the system cannot distinguish ``I found something relevant'' from ``I returned the least irrelevant thing I have.'' We introduce , a single-file system built on SQLite that addresses all three gaps. Our key insight is that adaptive RRF fusion -- adjusting vector and keyword weights per query using IDF analysis -- combined with MMR deduplication and distance-based relevance signaling, produces retrieval quality competitive with published baselines on standard benchmarks. Our primary contributions are: a self-supervised embedding refinement method exploiting vector/FTS disagreement, yielding a 33M-parameter pipeline that achieves NDCG comparable to or exceeding published ColBERTv2 results on 3 of 5 BEIR datasets with zero human labels, under different preprocessing conditions ; adaptive RRF with IDF-based per-query weight adjustment, validated on 5 BEIR datasets; a documented negative result showing that post-RRF scoring strategies fail to improve retrieval quality; and a production-grade substrate with integrity checking, schema versioning, and ranking diagnostics. Evaluation on BEIR revealed this assumption fails on long, semantically-rich queries: on ArguAna , fixed weights score NDCG@10 = 0.3599 vs 0.4370 for adaptive IDF -- a 17.6\ Adaptive RRF computes per-query weights using the mean IDF of Porter-stemmed query terms via a sigmoid function. High IDF boosts FTS weight, directing the system toward exact keyword matching. Low IDF boosts vector weight, relying on semantic similarity. The IDF vocabulary is built lazily from SQLite's fts5vocab virtual table on first search and cached for the lifetime of the process; subsequent lookups are dictionary accesses with negligible overhead compared to the embedding call. The cache is invalidated on writes . Additionally, long queries relax the distance cutoff from 1.15x to 5.0x the best match distance.
