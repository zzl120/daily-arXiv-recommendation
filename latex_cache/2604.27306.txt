A primary challenge for deployed RAG systems is maintaining correctness as the underlying corpus evolves~. In dynamic domains such as software documentation, news, or law, updates occur frequently~. Standard passage-based systems may retrieve conflicting information from different versions of a document. The generator might then merge outdated and current details, producing confident but incorrect answers. This issue arises from retrieving valid-looking but temporally inconsistent evidence. Recent research suggests that retrieving finer-grained units, such as propositions or triplets, improves relevance and reduces token costs~, and that augmenting RAG with user-centric agents yields further gains on personalised tasks~. However, existing implementations store these units as static text or embeddings, lacking metadata to capture temporal validity or source disagreement. Simultaneously, evaluation methodologies are shifting toward nugget-based scoring, where a ``nugget'' is defined as an atomic, self-contained fact~. This trend creates a design misalignment: systems are evaluated on precise atomic facts but retrieve unmanaged, multi-fact passages. Consequently, retrieval systems lack mechanisms to explicitly track or update the specific facts required for accurate generation. We propose bridging this gap with , an AI-governed index where retrieval units are not merely strings, but . We formally define a nugget not as a text span, but as a composite record $N = $, containing the atomic fact $f$, its temporal validity $v$, and its epistemic status $e$ . This data model relies on three properties to ensure maintenance. First, every nugget is assigned a ~, $[t_s, t_e)$, defining when the fact is true, rather than when it was written. Second, we explicitly model epistemic uncertainty through a lifecycle state: for current consensus, for superseded facts, and for unresolved disagreement. Third, nuggets maintain provenance links to source spans, grounding these validity judgments in evidence. At query time, filters the index to retain only nuggets that are and valid for the specific query time. nuggets are excluded, while items can be explicitly flagged. This filtering occurs before ranking, ensuring the generator processes only consistent and valid evidence. We make the following contributions: We formalize RAG maintenance as a retrieval task and identify the discrepancy between nugget evaluation and retrieval. We introduce the data model, which integrates validity intervals and lifecycle states, alongside algorithms for index construction and conflict resolution. We propose a governed retrieval pipeline that filters by temporal validity and lifecycle states before ranking. Ablations reveal that sparse retrieval outperforms dense on atomic nuggets, enabling sub-millisecond latency without embedding inference. Standard RAG systems typically retrieve passages ranging from 100 to 200 tokens and concatenate them to form a prompt for the generator~. While this approach is straightforward, it suffers from inherent limitations. Passages frequently contain a mixture of relevant and irrelevant content, and multiple passages may redundantly state the same information. Furthermore, long contexts increase the computational cost of generation~. To address these issues, recent research has explored retrieving finer-grained units. For instance, extract propositions—single-sentence atomic statements—and perform retrieval at this level, reporting improved precision. Similarly, triplet-based approaches extract subject-relation-object tuples and retrieve over these structures~. These methods succeed in improving relevance density and reducing prompt length. However, such atomic units are generally stored as plain text strings or embeddings without additional structure. They lack metadata indicating when a fact is valid or whether it has been superseded. Consequently, if two propositions contradict one another, the system lacks the mechanism to detect or resolve the conflict, potentially retrieving both and passing the contradiction to the generator. Nugget-based evaluation originated in the TREC QA tracks, where system answers were scored based on their coverage of ``information nuggets''—atomic facts that a complete answer should contain~. This method offers greater robustness than string matching because it credits semantically equivalent answers even when their surface forms differ. Recent advancements have scaled nugget-based evaluation through the use of LLM-based autograders~. Given a candidate answer and a set of gold nuggets, an autograder determines which nuggets the answer successfully covers, enabling evaluation on large benchmarks without requiring manual annotation for every response. The reliance on nugget coverage offers an opportunity to align retrieval design with evaluation by targeting nuggets directly. This alignment aids in diagnosing failures and enables the optimization of retrieval models for nugget recall. A parallel line of work uses nugget-like atomic units inside the retrieval pipeline rather than only at evaluation time. GINGER~ grounds response generation in atomic information nuggets retrieved per query; proposition-style retrievers~ index single-sentence propositions and report precision gains on factoid QA; triplet-driven retrievers~ index subject--relation--object tuples for RAG. These approaches share our atomic-unit choice but treat units as static text, omitting the validity interval and lifecycle state we maintain. On the temporal knowledge graph side, hybrid structural--semantic frameworks~ model the evolution of relations between entities over time and answer queries against curated graphs that combine structural and semantic signals to capture historical patterns; these systems rely on a fixed schema and pre-existing graph structure. sits between these two lines: we adopt the atomic-unit retrieval substrate of nugget-IR while importing the validity-interval discipline of temporal KGs into the retrieval index itself, with the schema discovered from the target corpus rather than authored in advance. Temporal Information Retrieval incorporates time into retrieval processes. Document-level methods use publication timestamps to rank by recency or filter within time windows~. Query-level approaches identify temporal intent to adjust retrieval strategies~. In the context of RAG, the core challenge is temporal validity. A document published in 2023 may discuss events from 2020, while a document from 2018 may state a fact that remains true until 2022. Document timestamps do not capture these nuances; validity information is required at the fact level. Temporal question answering benchmarks evaluate whether systems provide answers appropriate for a specific query time~. Systems that retrieve mixed-time evidence typically perform poorly on these evaluations. Knowledge bases like Wikidata store facts with qualifiers such as temporal constraints and references~. Wikidata statements can also be marked with ranks, such as preferred or deprecated, to indicate currency. Although this model is conceptually close to our proposed approach, Wikidata relies on a fixed ontology and formal schema. Populating the base requires entity linking and relation extraction against that specific schema, and querying requires structured languages like SPARQL~. These requirements limit the applicability of such systems to open-domain text corpora where facts do not conform to a predefined structure. Alternatively, GraphRAG approaches construct graphs over documents and utilize graph traversal or graph neural networks for evidence retrieval~. Although these methods enable multi-hop reasoning by capturing relationships, they require expensive graph construction and suffer from increased inference latency. An LLM-based extractor decomposes each window into atomic statements. The LLM extracts facts from documents without cross-document awareness. The system identifies conflicts downstream when multiple nuggets share the same key $ $ but have different values and overlapping validity intervals. This design provides robustness to LLM inconsistency. A hallucinated fact from a single document lacks the evidence redundancy required to deprecate existing knowledge and is isolated as . The existing consensus remains until the new claim accumulates sufficient corroboration. Following validity inference, we integrate candidates into the index via deduplication and state assignment: If the candidate matches an existing key $ $, we compare object values. Exact matches or near-duplicates are merged. For example, ``Google's CEO Sundar Pichai'' and ``Pichai, leader of Google'' resolve to the same key and similar values, allowing storage as a single nugget with accumulated evidence. If keys match but values differ and the relation is functional, we examine temporal overlap. Non-overlapping intervals indicate succession , yielding a new nugget. Overlapping intervals trigger conflict resolution. Multi-valued relations accept concurrent values. For functional relations: if the newer candidate has $ 2$ independent sources, the existing nugget is marked ; if the existing nugget has superior evidence, the candidate is deprecated; ambiguous cases mark both , requiring $ 3$ sources for resolution.
