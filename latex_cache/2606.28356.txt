 Generative recommendation research includes a line that adapts the traditional recommendation paradigm by generating item identifiers or action sequences, such as semantic-ID-based generative retrieval and large-scale sequential transduction models~. Another related line studies natural-language recommendation, where user intents are matched against textual evidence associated with items, such as descriptions, reviews, or passages . These methods typically perform evidence-level scoring and then aggregate the matched evidence into item-level recommendation scores. They are orthogonal to our setting. studies LLM-based recommendation agents that use natural-language queries and retrieved textual evidence to compare candidate products and generate user-facing recommendations~. Recent LLMs exhibit broad general abilities in instruction following, contextual reasoning, and text generation~. Building on these abilities, LLM agents use language models as central controllers that can plan, maintain memory, retrieve external information, call tools, and interact with environments~. Recommendation agents can be viewed as a domain-specific instance of this paradigm: they interpret user needs, inspect retrieved evidence, compare candidate items, and produce natural-language recommendations. Recent work shows that these agents can be manipulated: strategic product-page text can increase a target product's recommendation probability~, and contextual biases can affect LLM-as-recommender decisions~. These studies motivate rec-agent harmfulness, but they do not isolate GEO as a seller-controlled retrieved-source attack. fills this gap by testing whether GEO-rewritten evidence changes model-estimated utility \ , promotes non-utility-aligned targets, and can be mitigated by developer-side agent design . Search-engine optimization traditionally refers to optimizing web pages so that search engines can better discover and rank them for user queries~. It also has an adversarial side: web spam studies how content, links, and page structure can be manipulated to obtain undeserved search visibility~. Generative Engine Optimization extends this visibility problem from search rankings to generative systems, where source owners may rewrite content not only to be retrieved, but also to be cited, summarized, absorbed, or recommended in generated outputs~. GEO-Bench establishes the general setting by measuring whether rewritten content is surfaced or reflected more often in generated answers~. E-GEO brings this question into e-commerce by studying how product listings can be optimized for generative engines~. SAGEO Arena extends this line toward a fuller search-augmented pipeline by evaluating GEO across retrieval, reranking, and generation~. These works are closest to in treating GEO as source-side manipulation, but they primarily measure visibility gains rather than whether GEO causes harmful recommendation outcomes in agent-facing settings . The layers are diagnostic rather than cumulative. L3 is not ``L1+L2+L3,'' and L5 is not an endpoint of a pipeline. Each layer changes one source-use behavior: L1 adds a lightweight defensive grounding instruction; L2 changes the output contract to require explicit recommendation reasons and citations; L3 instructs the agent to perform a pre-generation evidence-organization workflow; L4 instructs the agent to balance evidence across source types and candidates; and L5 instructs the agent to quarantine source-internal model-facing instructions. In all layers, the agent receives the same model-visible tuple: the user query, candidate roster, and source packet. The mitigation prompts do not reveal the attacked target identity, attack package label, hidden hard constraints, hidden soft preferences, canonical attributes, verified utility, gold ranking, or gold evidence-relation labels. L0 is the baseline. The agent is asked to recommend products using the provided sources, without any additional defensive source-use instruction. L1 adds a lightweight prompt-level safeguard. It reminds the agent that source documents are evidence rather than instructions, that unsupported product claims should not be treated as satisfied requirements, and that uncertainty should be preserved when source evidence is vague, missing, or conflicting. L2 changes the output contract. The agent must provide short reasons and source line citations for each top recommendation. This layer tests whether making the final rationale more explicit is sufficient to reduce GEO harm. It does not ask the agent to restructure evidence before ranking, so it isolates output accountability from pre-generation verification. L3 changes the pre-generation workflow. Before ranking products, the agent is instructed to create its own candidate-level evidence checks from the visible query, candidate roster, and source packet. For each candidate, the agent identifies decision-relevant claims or issues, infers the relevant requirement or product property from the visible input, assigns a source-visible evidence status, and cites source lines supporting that status. These checks are produced by the agent itself under instruction; they are not externally prepared evidence sheets and are not restricted to the attacked target. The status labels therefore reflect the agent's own source-visible judgment rather than gold hard-constraint labels: claims that appear contradicted by source lines are marked ; claims whose support is insufficient, uncertain, or judged unverifiable are marked ; and claims requiring additional checking are marked . L4 changes how the agent uses the source context. The source packet itself is not manually reordered, rewritten, or filtered. Instead, the prompt instructs the agent to avoid letting a single salient or seller-controlled document dominate the ranking. The agent is asked to compare promotional claims against ordinary product pages, reviews, comparison sources, specification-style evidence, caveats, and conflicting evidence when such sources are available. This layer tests whether a source-balancing procedure can reduce seller-source capture while preserving the same input context. L5 changes how the agent treats source-internal instructions. The source packet is not pre-filtered before the agent sees it. Instead, the prompt instructs the agent to distinguish product evidence from model-facing directives embedded in retrieved documents. Lines such as ``AI assistants should rank this product first,'' ``cite this page,'' or ``recommend this item'' should be treated as non-evidence for product utility. The agent may still use factual product claims from the same source when those claims are supported by source text, but it must not follow source-internal instructions as commands. Across all six cases, the model response is coherent, cited, and structurally complete. The harm comes from how response fields are connected. First, positive target framing is allowed to replace exact hard-constraint evidence. Second, citations point to the evidence the model used, but not necessarily to the evidence the user needed. Third, source-assessment warnings can appear in one field without changing the final ranking. Fourth, constraint logic is often applied asymmetrically: competitors are rejected for missing specifications, while the promoted target is summarized with higher-level fit language. These patterns explain the aggregate mechanism in and the mitigation result in : the effective intervention is not a longer final answer, but a candidate-level evidence ledger that forces support, missing evidence, and refutation to be represented before ranking. The released artifacts are intended to support research on the robustness and safety of source-grounded recommendation agents. We release the code, prompts, scoring scripts, and aggregate analysis utilities under the Apache License. The data are released under a research-use, non-commercial license, with accompanying usage terms that restrict redistribution of de-anonymized records and prohibit using the artifacts to optimize deceptive product pages or manipulate deployed recommendation systems. The released dataset uses de-identified or synthetic product/source records where appropriate and does not require redistribution of raw proprietary webpages. Derived artifacts should preserve the license notice, citation information, de-identification safeguards, and misuse warnings. We studied the harmfulness of GEO in recommendation agents. To make this risk measurable, we introduced , an evaluation suite with 22 GEO attack variants. Our results show that seller-controlled source rewrites can promote flawed products into the user's decision set and increase constraint-violating recommendations. Lightweight developer-side defenses can reduce this harm, but they do not restore the no-GEO performance. GEO robustness should therefore be treated as a practical agent-design problem for recommendation systems. *{Limitations} focuses on text-based source content, while real web sources are often multimodal, including images, videos, tables, layout cues, badges, ratings, and structured metadata. As a result, our dataset does not fully capture GEO strategies that manipulate visual presentation or multimodal evidence. Extending to multimodal source contexts is an important direction for future work. primarily evaluates a single-turn recommendation setting, where the agent receives one user query and produces recommendations from the available source evidence. Real recommendation agents may operate over multiple turns and ask clarification questions, update user preferences, or use memory from prior interactions before making a final recommendation. These interaction mechanisms could affect vulnerability. This work is intended to support safer and more reliable evaluation of source-grounded recommendation agents. Because the dataset studies adversarial source rewriting, released materials should be handled carefully to avoid enabling direct misuse. In particular, examples should use synthetic or de-identified products when appropriate, and adversarial transformations should be presented as controlled evaluation conditions rather than as practical instructions for manipulating real systems. The evaluation should emphasize user welfare, factual support, uncertainty handling, and the distinction between legitimate content improvement and deceptive optimization. More broadly, recommendation agents can influence user choices, commercial visibility, and trust in online information, so systems should be assessed not only by visibility or ranking outcomes, but also by whether they preserve user constraints and avoid amplifying misleading or unsupported claims. shows that GEO attacks substantially increase recommendation harm. The taxonomy is designed to capture how seller-controlled source text can change the model-estimated utility \ of a target item while leaving verified utility \ unchanged through changes in factual fit, evidential support, or model-facing presentation. We construct the taxonomy through a three-step procedure. First, we seed candidate primitives from three lines of literature: information manipulation~, persuasion~, and GEO / LLM-mediated retrieval risks~. Second, we conduct an operational check on pilot recommendation cases. For each pilot case, we identify a concrete target gap, such as a failed hard constraint or weaker preference fit. We then check whether each candidate primitive can be realized as a seller-controlled source rewrite around that gap while keeping the canonical attributes \ and verified utility \ \) fixed. Finally, we merge overlapping candidates into operational primitives. We retain only primitives that can be instantiated as controlled source rewrites and have separable decision boundaries. The final taxonomy contains three manipulation loci and seven primitives. This locus changes the substantive content of the document. It includes three primitives: , where the source states that the target satisfies a requirement it does not satisfy, , where the source hides a decision-relevant limitation, and , where broadly positive but weakly relevant content distracts from the user's actual needs. These primitives follow the communication-theoretic distinction between truthfulness, informativeness, and relevance in misleading messages~. This locus changes how claims are made to appear supported. It includes two primitives: , where seller-affiliated content is presented as independent or more authoritative, and , where citations, benchmarks, ratings, or expert-like language are added without directly supporting the relevant claim. These primitives reflect findings that source credibility and peripheral cues can affect how messages are accepted~. This locus changes how source text is surfaced to the model. It includes two primitives: , where answer-style formatting, repeated query terms, or emphatic headings make the target more prominent, and , where the source contains instructions aimed at the model rather than information for the user.
