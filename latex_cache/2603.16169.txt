Large language models have demonstrated remarkable capabilities across a wide range of natural language tasks, yet they remain susceptible to hallucinations --- generating factually incorrect content with apparent confidence . Retrieval-Augmented Generation addresses this limitation by grounding generation in externally retrieved documents. However, RAG assumes that retrieved documents are relevant, which is frequently not the case in practice. Corrective Retrieval Augmented Generation addresses this assumption by introducing a lightweight retrieval evaluator that assesses document quality and triggers one of three corrective actions: , , or . When retrieval quality is low, CRAG falls back to web search to obtain better context. This corrective mechanism has been shown to significantly improve generation accuracy across multiple benchmarks. Despite CRAG's strong results, reproducing the original system is difficult. The original implementation relies on the Google Search API , proprietary LLaMA-2 fine-tuned weights, and deprecated OpenAI API calls. These barriers prevent researchers from building on this work without significant resources. In this paper, we make three contributions: We present a fully open-source reproduction of CRAG, replacing all proprietary components with free alternatives: the Wikipedia API for web search and Phi-3-mini-4k-instruct as the generator. We evaluate our reproduction on two datasets --- PopQA and ARC-Challenge --- demonstrating that open-source components achieve comparable performance. We contribute the first explainability analysis of CRAG's T5-based retrieval evaluator using SHAP , identifying that the evaluator relies primarily on named entity alignment and exhibits systematic domain transfer failure on science questions. Retrieval-Augmented Generation enhances language model generation by prepending retrieved documents to the input. Given a query $x$ and a corpus $$, a retriever $R$ returns the top-$k$ documents $D = \{d_1, , d_k\}$, and a generator $G$ produces the output $y$ conditioned on both $x$ and $D$. While RAG significantly improves factual accuracy, it is sensitive to retrieval quality --- irrelevant documents can mislead the generator and worsen performance compared to no retrieval at all . CRAG introduces a retrieval evaluator $E$ that scores each retrieved document against the query. Based on these scores, one of three actions is triggered: : At least one document scores above an upper threshold $ ^+$. The documents are refined using a decompose-then-recompose algorithm to extract relevant knowledge strips. : All documents score below a lower threshold $ ^-$. Documents are discarded and web search is used to find external knowledge. : Scores fall between $ ^-$ and $ ^+$. Both internal and external knowledge are combined. The retrieval evaluator is a fine-tuned T5-large model that takes a question-document pair as input and outputs a scalar relevance score in $ $. For PopQA, the original paper sets $ ^+ = 0.59$ and $ ^- = -0.99$. SHAP is a game-theoretic framework for explaining model predictions. For text models, SHAP assigns each token an attribution value representing its contribution to the model output. We use the with a text masker to explain the T5 retrieval evaluator's scoring decisions at the token level. Several works have explored making RAG more robust to retrieval failures. Shi et al.\ demonstrate that irrelevant context can significantly harm LLM generation, motivating corrective approaches. Self-RAG trains LLMs end-to-end to generate special reflection tokens that decide when to retrieve and how to critique retrieved passages --- a fundamentally different approach from CRAG's plug-and-play external evaluator. FLARE actively decides when to retrieve based on generation confidence, keeping the generator frozen and adding a lightweight external T5 evaluator that makes it easy to deploy without retraining. Our work reproduces CRAG with open-source components and adds the first token-level explainability analysis of its retrieval evaluator. While RAG systems have been widely studied, explainability of their internal components remains underexplored. Prior work has applied attention-based analysis to understand retrieval , but SHAP-based token attribution for retrieval evaluators has not been studied. Our work contributes to this direction by revealing that the T5 evaluator functions as an entity alignment detector rather than a semantic relevance judge, which has direct implications for its domain generalization.
