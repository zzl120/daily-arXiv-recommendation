As the final stage of this cascade, reranking~ takes a request-specific local candidate set produced by earlier stages and arranges the items into an ordered slate for user exposure. Unlike retrieval and ranking, which often evaluate items independently, reranking must model , including substitution, complementarity, and diversity. For example, placing two near-identical items next to each other can reduce slate diversity, while placing complementary items together may improve user satisfaction. The central challenge in reranking is therefore to generate an ordered slate that captures these interactions while respecting the candidate pool returned by earlier stages. Recent work frames reranking as a sequence generation problem $P $, where the model constructs an exposure slate $S$ from the user context $u$ and a request-specific candidate pool $C$. The majority follow two paradigms: ~ factorize the slate in a fixed left-to-right order and condition each item based on previously generated items. This factorization captures dependencies among exposed items, but the causal attention mask prevents each slot from using information from later slots, and an early mistake can propagate irreversibly to subsequent slots~. ~ predict all slots in parallel and avoid this error propagation, but they usually rely on a slot-wise independence assumption that weakens dependency modeling within the exposure slate. A recent diffusion-based reranker~ introduces iterative slate refinement, but it operates after an initial complete list has already been given. Thus, it corresponds more closely to refinement over a starting slate than to adaptive slate construction from uncertain positions. As illustrated in Figure~, existing generative rerankers therefore expose a tradeoff between and . This raises the central question of this paper: {figure/intro.pdf} We answer this question with , a unified list-wise reranking framework that constructs slates via confidence-ordered iterative denoising. Starting from an all mask slate, ~uses bidirectional slate modeling to score candidate assignments for each unfilled position, commits the most confident position-item assignment, and repeats this process until the slate is complete. This design lets every prediction use the current slate context while avoiding a fixed left-to-right commitment order. Under different inference schedules, the same framework recovers both paradigms: 1) Forcing the denoising order to follow left-to-right gives an AR reranker, while 2) collapsing the process into all-slot parallel generation steps gives an NAR reranker. Thus, ~provides a unified inference view that connects AR decoding, NAR decoding, and confidence-ordered iterative decoding. To instantiate ~for reranking, the diffusion process must also match the item-level modeling and candidate-constrained nature of the task. Mechanically, we introduce the Task Grounded Diffusion interface, which contains two modules: 1) aggregates each item's semantic representations into a single item embedding, so each denoising position corresponds to one item rather than a sequence of tokens. 2) replaces vocabulary level decoding with dot product scoring against the local candidate pool, so each prediction is made directly over legal candidates for the current request. Reranking is the final stage of a modern multi-stage recommender pipeline, and it determines the list that is ultimately shown to the user~. Existing neural rerankers can be broadly grouped into two paradigms: Generator-only and Generator-Evaluator. Generator-only methods use a single model to produce the slate directly~. DLCM~ refines pointwise scores with a recurrent model that captures full list context; Seq2Slate~ formulates slate construction as autoregressive decoding with a pointer network; PRM~ replaces recurrent encoding with self-attention so that each candidate can interact with all other candidates in one pass; and GoalRank~ revisits this paradigm with a large generator trained by group relative optimization. Generator-Evaluator methods~ separate list proposal from list scoring. PIER~ combines a SimHash-based permutation generator with a context-aware evaluator, NAR4Rec~ generates slot assignments in parallel with unlikelihood training and contrastive decoding, and MG-E~ uses multiple complementary generators to enlarge the candidate list space. Despite these design differences, both paradigms still rely on a fixed decoding order, which is either autoregressive and vulnerable to error propagation or fully parallel and therefore limited in modeling dependencies among target positions. ~instead formulates reranking as masked discrete diffusion over the slate and commits slot to candidate assignments in order of their confidence, which unifies list generation and refinement within a unified framework and yields bidirectional slate attention together with an adaptive any order decoding schedule. The closest prior work to ~is DCDR~, which is equivalent to the refinement stage of ~applied to an external given initial list. As a result, its output is anchored to that initial list and its quality is tightly bounded by the quality of the starting point. Generative recommendation reformulates recommendation as a sequence generation problem over learned item identifiers, where the model predicts the next item token by token or through iterative denoising instead of assigning a pointwise matching score~. Progress in this direction can be organized along three axes. Tokenization. Recent work replaces atomic item identifiers with content-informed representations. P5~ converts user item interactions, item metadata, and related signals into natural language sequences within a unified text-to-text framework, while TIGER~ learns hierarchical semantic identifiers from pretrained item embeddings through residual vector quantization. Architecture. Encoder-decoder models such as M6-Rec~ use a unified pretrained language model to support retrieval, ranking, and explanation, whereas decoder-only models emphasize industrial scaling, with HSTU~ proposing a high-throughput sequential transduction architecture for long and dynamic user histories. More recently, LLaDA-Rec~ replaces left-to-right decoding with a masked discrete diffusion backbone~ that generates the semantic identifier of the next item. Optimization. Beyond next token prediction, TallRec~ shows that instruction tuning with small amounts of recommendation data can improve few-shot recommendation and cross-domain transfer.
