Modern CLI coding agents, spanning research systems such as SWE-agent~ and OpenHands~ as well as widely deployed products including Claude Code, Codex, and OpenCode, have become a dominant interface for repository-level software engineering. They expose the LLM to the codebase through a small, fixed action set centered on shell-level tools, a minimalist design that pairs naturally with the identifier-centric nature of code and underpins strong performance on various coding tasks. However, this lexical-only interface leaves structural dependencies invisible unless surfaced through additional reasoning steps. Agentic systems such as SWE-agent~, OpenHands~, and OrcaLoca~ localize via iterative shell-level search, while non-agentic pipelines such as Agentless~, CodePlan~, and HiLoRM~ use static multi-stage retrieval; RepoNavigator~ and SweRank+~ further learn exploration and ranking policies. These methods remain predominantly lexical, motivating work that augments localization with explicit graph retrieval over the repository. Graph retrieval~ is most effective for evidence that is sparse, multi-hop, and relationally typed~, the regime of code repositories. Code-focused approaches~ differ in how graph evidence reaches the agent: ~ consume offline-built graphs but lack adaptive exploration; ~ expose graph queries as new agent actions but fragment the search loop; ~ steer exploration via graph structure but tightly couple traversal with agent control. LARGER instead delivers graph evidence through the agent's existing lexical channel, jointly addressing graph quality and retrieval efficiency by design without breaking the CLI agent regime. Effective code localization must exploit structural dependencies without violating the agent's stepwise interaction constraints. These requirements lead to several key challenges: How to represent repository structure so that task-relevant dependencies are reachable through compact local neighborhoods. How to expose useful structural evidence without incurring the overhead of global graph traversal or excessive context growth. How to adapt retrieval decisions as the agent’s context evolves during interaction. We address these challenges by coupling lexical anchoring with localized graph expansion, enabling efficient exploration while progressively incorporating structural evidence. Constructing $G_{c_0}$ from scratch parses every file in the repository at the reference commit $c_0$ and runs the heterogeneous extraction, confidence weighting, and community partitioning passes, incurring cost $O $ in the resulting graph size plus parser overhead per file.
