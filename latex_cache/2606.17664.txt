Like DPO, Temporal Retrieval Preference Optimization forms $ $ as a pairwise preference pair over a timestamped document corpus given a query $Q$, time-aligned document , and unaligned document . The goal of TRPO is to prefer temporally aligned document over the misaligned given a query and minimizing the TRPO loss function in Eq.~. $_{}$ is based on the Bradley-Terry model. While DPO aligns model score with human-labeled preference, TRPO aligns score with temporal relevance with an implicit signal derived from corpus-level differences . In this view, TRPO requires working under the following three conditions. There must be a certain temporal preference gap between aligned and misaligned document $ > $ when $t' t$ where $ $ is a minimum gap required. If the actual document update with temporal change is too small relative to noise, TRPO learning could be unstable. To handle this issue, we comprise a temporally distinct document collection by filtering the dataset in Appendix~. Aligned and misaligned temporal corpora should cover a similar set of topics, so semantic similarity may remain high and the only difference is the timestamp and the document content at that timestamp. Encoder should have sufficient capacity to represent latent temporal signal as well as semantic similarity. Under these conditions, TRPO encourages the model to rank temporally aligned documents higher. The resulting scoring function $S_ $ is expected to approximate one that reflects temporal alignment between query and document. This mirrors the theoretical guarantees for DPO by replacing ``generation quality'' with ``temporal relevance'' as the underlying reward~. To sum up, TRPO is a preference alignment variant, where preferences are defined by temporal grounding between a versioned corpus. This generalizes preference learning to the temporal dimension.
