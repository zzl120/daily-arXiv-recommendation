Traditional information retrieval systems often overlook user-specific needs and return identical results for the same query, despite differences in users' knowledge, preferences, and objectives. We integrate external semantic resources, namely a domain ontology and user profiles, into the retrieval process. Documents, queries, and user profiles are represented as vectors of weighted concepts. The ontology applies a concept-weighting mechanism that emphasizes highly specific concepts, as lower-level nodes in the hierarchy provide more precise and targeted information. Relevance is assessed using semantic similarity measures that capture conceptual relationships beyond keyword matching, enabling personalized and fine-grained matching among user profiles, queries, and documents. Experimental results show that combining ontologies with user profiles improves retrieval effectiveness, achieving higher precision and recall than keyword-based approaches. Overall, the proposed framework enhances the relevance and adaptability of XML search results, supporting more user-centered retrieval. Information retrieval User profile Domain ontology Personalized search XML Vector space model Information Retrieval is a fundamental field of computer science, aiming to identify and extract documents from large collections based on their relevance to a user’s information needs, typically expressed as textual queries . An Information Retrieval System generally encompasses three core tasks: representing document content, modeling the user’s query or information need, and comparing these representations to compute a relevance score. One of the enduring challenges in IR lies in accurately modeling relevance so as to deliver the most meaningful and useful results . The rapid digitization of content and advances in web technologies have led to exponential growth in available data. Over time, document formats have evolved from flat, unstructured text to semi-structured representations such as SGML and, more prominently, XML---a widely adopted standard for data representation. XML is particularly well suited for IR because it combines structural richness with descriptive content, enabling fine-grained and personalized access to information . Despite notable progress in indexing and querying techniques, personalized search systems still face important limitations. Many approaches rely on user profile models that are either overly generic or insufficiently expressive, failing to capture the complexity of user preferences . Moreover, the semantic integration between user preferences and document structure is often shallow, neglecting the hierarchical granularity of XML documents and the richness of conceptual relationships . Most existing work also focuses primarily on the profiling phase without providing mechanisms for the continuous adaptation of profiles as user queries evolve . To address these gaps, we propose an approach that integrates semantic user profile modeling via a domain ontology with structural exploitation of XML documents. Personalization is achieved by constructing a user profile that reflects individual interests and incorporating it into the search process to improve result relevance. Our system represents user interests as weighted ontology-based vectors, enabling fine semantic interpretation of queries. Profiles are dynamically updated after each user interaction through a progressive reinforcement mechanism for the most relevant concepts. The choice of XML as the core document representation is motivated by its structured nature and semantic richness, which are key features for fine-grained, personalized IR. Its hierarchical organization supports precise content modeling, while its compatibility with ontology languages such as OWL facilitates semantic integration . XML also offers a flexible framework for representing both data and user preferences, making it a strong candidate for adaptive and intelligent search systems. While other formats such as JSON, YAML, Markdown, or NoSQL data models are prevalent in modern development , they have limitations in representing rich hierarchies and semantic annotations. JSON, for instance, is efficient for API-based data transfer but lacks native typing, complex schema validation, and smooth integration with Semantic Web standards such as RDF or OWL . YAML and Markdown are even more minimalist, making them unsuitable for deep semantic structuring . NoSQL databases, including document-based and column-based systems, excel in scalability but do not provide standardized frameworks for fine-grained semantics, semantic validation, or automated reasoning . XML thus remains a strategic choice, combining explicit structure, W3C standardization, compatibility with semantic technologies, and flexible processing through XPath/XQuery. These features make it an ideal medium for personalized IR in complex, information-rich environments. Recent studies highlight XML’s potential in personalized search, leveraging profile extraction, semantic transformation, and structured indexing techniques . However, few offer a coherent integration of ontology, XML structure, and dynamic online profile adaptation. Our work addresses this gap by providing a unified framework that combines ontological representation, dynamic vectorization of user interests, and semantic exploitation of XML documents to improve the accuracy, personalization, and relevance of retrieved results. The main contributions of this paper are summarized as follows: We propose a personalized information retrieval framework for semi-structured XML documents that integrates a domain ontology with user profiles to improve result relevance. We represent documents, queries, and user profiles as ontology-based weighted concept vectors, and introduce a concept-weighting mechanism that emphasizes more specific ontology concepts. We design a personalized matching and ranking process based on semantic similarity, coupled with a dynamic user-profile updating mechanism that progressively reinforces concepts inferred from user interactions. We validate the proposed approach through experiments on an XML document collection, showing improved retrieval effectiveness compared to a non-weighted baseline, and we will make the source code and experimental data publicly available to support reproducibility. A further stream addresses personalization through user profiles, which capture contextual signals such as interests, preferences, goals, and interaction history . Existing approaches include XML-based preference/profile representations with keyword-oriented ranking and integrations of IR techniques with native XML databases for efficient structural access . In this paper, we bridge these directions by integrating a domain ontology and a dynamically updated user profile, representing documents, queries, and profiles as weighted concept vectors to enable fine-grained semantic matching for personalized XML retrieval. To emphasize specific concepts , we assign weights to ontology concepts. We assume that the total weight of the ontology equals one , i.e., it is the sum of the weights of all its concepts. The weight is distributed across ontology levels so that concepts at depth $i$ receive higher weights than those at depth $i-1$. Coefficients are propagated toward leaf concepts, starting from the highest depth, and increased only when moving from depth $i$ to a deeper depth $i+1$. The experiments target personalized information retrieval in semi-structured XML documents. Although reference collections such as INEX~, TREC~, and CLEF~ are widely used, they do not fully meet our needs. In particular, while INEX is XML-oriented, it is difficult to exploit for reliably mapping documents to the concepts defined in our ontology. Moreover, INEX officially ceased in 2015, which limits its suitability for recent personalized settings~. More generally, these collections do not explicitly address personalization or the dynamic evolution of user preferences in semi-structured environments. For these reasons, we built our own test database. It contains 400 XML documents covering several computer science topics, organized according to the concepts of our domain ontology. We also generated 300 queries by selecting ontology concepts and using their keywords to simulate different information needs. We also observe that more specific ontology concepts can yield higher effectiveness than broader concepts , which is consistent with the ontology weighting strategy that favors deeper, more specific concepts. This paper presented a personalized XML information retrieval approach that combines a domain ontology with a dynamically updated user interest vector. By weighting ontology concepts and continuously adapting profile weights based on user queries, the system exploits explicit semantic structure while accommodating evolving preferences. Experimental results, including F1-scores across different user profiles, indicate that the approach improves retrieval effectiveness, particularly for specific concepts supported by the ontology hierarchy. Future work will focus on validating the approach on larger and more heterogeneous XML collections and on extending it beyond the computer science domain. We also plan to investigate neural methods to refine user interest vectors from both explicit and implicit interaction signals, and to explore automatic ontology enrichment from XML corpora.
