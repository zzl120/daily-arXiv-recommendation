In this demo paper, we present , an LLM-agent-based conversational recommender system explicitly designed to promote sustainable tourism practices. Unlike traditional chatbots or chat-based travel assistants, TRACE incorporates sustainability objectives from the ground up. The system prioritizes recommendations such as less-visited or emerging destinations over overcrowded hotspots, and favors lower-impact transportation options whenever feasible. Crucially, these sustainability signals are not hard-coded defaults but are inferred through interaction. The primary goal of this work is to build a state-of-the-art recommender system in terms of predictive accuracy or ranking performance. Instead, we aim to explore the design space and research opportunities enabled by multi-agent LLM architectures for sustainable conversational recommendation. In particular, we focus on how such systems can be structured to users toward more sustainable travel choices without violating their explicit preferences or degrading user trust. TRACE achieves this through a modular, agentic design. Dedicated agents generate targeted clarifying questions to elicit latent sustainability preferences, construct structured user personas, and produce recommendations that jointly optimize for relevance and sustainability signals. Conversational recommender systems have been widely studied for interactive preference elicitation and natural language decision support. Recent LLM-powered, multi-agent approaches leverage specialized agents for user modeling, planning, and explanation, enabling collaborative recommendation and flexible reasoning~. However, these systems primarily focus on accuracy and engagement, leaving sustainability and behavior change as secondary concerns~. Although recent work~ has explored multi-agent frameworks to balance relevance and sustainability in tourism recommendations, these approaches do not explicitly incorporate conversational interactions or leverage explanations and counterfactuals to nudge user behavior, and they remain primarily research prototypes with limited production readiness. Explainable and counterfactual recommendation research further explores transparency and bias mitigation, often via minimally contrastive explanations~. Relatedly, clarifying questions have been investigated as a way to uncover hidden or incomplete user preferences~, with taxonomies spanning vague, spatial, and temporal dimensions~ and LLM-based dialogue updates supporting personalization~. We formalize the TRACE Conversational Recommender System as a sequential pipeline of transformations over the state space of a user session. Let $q $ denote the initial natural language query. The system proceeds through the following stages. The Clarifying Question Agent $A_{}$ generates a set of targeted questions $C = \{c_1, c_2, , c_n\}$ to elicit latent user preferences related to sustainable travel: C = A_{} . The primary goal of the Clarifying Question Agent is to elicit the user's sustainability preferences indirectly and their willingness to compromise, for example, by asking whether they would be willing to travel to a lesser-known place rather than a popular destination, thereby improving intent detection. However, if the user's initial query is vague , then $A_{}$ is also instructed to ask the user about general travel aspects such as their budget and interests. The user's initial query is provided as input to $A_{}$, which generates up to 5 clarifying questions, which are then shown to the user one at a time. Given the user responses $ $ to these questions, the Intent Classification Agent $A_{}$ maps the interaction into a structured $$ and a vector representing the user’s across sustainability dimensions such as emissions, congestion, and seasonality: = A_{} . The generated and are provided to the recommender to obtain personalized, context-aware recommendations. The system constructs two candidate recommendation sets. The baseline set $R_0$ is instructed to prioritize relevance solely to the original query $q$. In contrast, the sustainable set $R_1$ incorporates the inferred user persona and sustainability preferences from $A_{}$. Both sets are generated by the Recommender Agent $A_{}$, a Rec-LLM using few-shot prompting. In particular, $R_1$ is produced by prioritizing sustainability-related signals $S$ derived from the clarification responses $ $, while preserving alignment with the inferred persona $$: R_1 = A_{} . The Explanation Generation Agent $A_{}$ serves as the final decision-making component. It outputs a selected recommendation $r^*$, a persuasive explanation $E$, and a counterfactual alternative $r_{}$, explicitly conditioned on the inferred willingness to compromise: = A_{} . Each session proceeds as follows. Given their query, the user first responds to a set of clarifying questions to construct a travel persona. The system then generates two candidate recommendations: a baseline recommendation $R_0$ and a context-aware recommendation $R_1$. Based on the user’s willingness to compromise on sustainability, the Explanation Generation Agent determines which candidate is presented as the primary recommendation, $r^*$, and which is shown as the alternative, $r_{}$.
